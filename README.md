# Wisut Work Hub

ระบบหลังบ้านทีม สส. สำหรับจัดการประชุม รายงานประชุม งานค้าง ประเด็นงาน คอนเทนต์ หนังสือเข้า-ออก รายชื่อ และคลังไฟล์งาน

ออกแบบสำหรับงานของ **วิสุทธิ์ ตันตินันท์**
- สส.บัญชีรายชื่อ พรรคประชาชน
- กมธ. ศึกษาและติดตามการจัดทำงบประมาณ
- ประธานอนุกรรมาธิการปฏิรูปการจัดซื้อจัดจ้างภาครัฐ
- ทีมพรรคประชาชนต้านทุจริต

## ฟีเจอร์ในเวอร์ชันนี้

1. Login หลังบ้าน
2. Dashboard ภาพรวม
3. จัดการประชุม
   - นัดประชุม
   - วาระประชุม
   - ผู้เข้าร่วม
   - สรุปการประชุม
   - มติ/ข้อสั่งการ
4. งานค้าง / To-do
   - ผู้รับผิดชอบ
   - กำหนดส่ง
   - ความสำคัญ
   - ผูกกับประชุมหรือประเด็นงาน
5. ประเด็นงาน
   - จัดซื้อจัดจ้าง
   - งบประมาณ
   - TOR
   - ราคากลาง
   - บัญชีนวัตกรรม
   - Factor F
   - ระเบียบ
6. คลังไฟล์
   - อัปโหลดไฟล์
   - ผูกไฟล์กับประชุม/ประเด็น
   - ดาวน์โหลดไฟล์
7. คอนเทนต์
   - คลิปประจำสัปดาห์
   - โพสต์ชวนคนมาฟัง
   - ฟอร์มประกอบโพสต์
   - สรุปประชุมลงเพจ
8. หนังสือเข้า-ออก
9. รายชื่อเครือข่าย/หน่วยงาน
10. เพิ่มผู้ใช้งานทีม

## เทคโนโลยี

- Node.js
- Express
- SQLite ผ่าน `better-sqlite3`
- Multer สำหรับอัปโหลดไฟล์
- JWT cookie สำหรับ login
- HTML/CSS/Vanilla JS แบบไม่ต้อง build frontend

## วิธีติดตั้ง

ต้องมี Node.js รุ่น 20 ขึ้นไป

```bash
cd wisut-work-hub
npm install
cp .env.example .env
npm run dev
```

จากนั้นเปิด:

```text
http://localhost:3000
```

## บัญชีเริ่มต้น

ถ้าไม่เปลี่ยน `.env` ระบบจะสร้างบัญชีแรกให้อัตโนมัติ:

```text
อีเมล: admin@wisut.local
รหัสผ่าน: change-me-1234
```

> สำคัญ: ก่อนใช้จริง ให้แก้ `.env` โดยเฉพาะ `JWT_SECRET`, `ADMIN_EMAIL`, `ADMIN_PASSWORD`

## โครงสร้างไฟล์

```text
wisut-work-hub/
├── server.js          # แอปหลัก
├── package.json
├── .env.example
├── data/              # ฐานข้อมูล SQLite จะถูกสร้างที่นี่
├── uploads/           # ไฟล์อัปโหลด
└── public/
    ├── styles.css
    └── app.js
```

## Backup ข้อมูล

ควร backup 2 ส่วนนี้เป็นประจำ:

```text
data/workhub.db
uploads/
```

## Deploy ใช้งานจริง

### ทางเลือกง่ายสุด

ใช้ VPS หรือบริการ Node.js hosting เช่น Railway, Render, Fly.io หรือ DigitalOcean แล้วรัน:

```bash
npm install --omit=dev
npm start
```

### หมายเหตุเรื่อง Vercel

โปรเจกต์นี้เก็บไฟล์ในโฟลเดอร์ `uploads/` และใช้ SQLite ในไฟล์เดียว จึงเหมาะกับ VPS/Render/Railway มากกว่า Vercel เพราะ Vercel ไม่เหมาะกับ persistent local storage
